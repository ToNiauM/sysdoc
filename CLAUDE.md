# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

SysDoc is an AI-agnostic workflow for comparative technical and legal analysis of Brazilian public procurement documents (ETP and TR). Any AI (Claude, GPT, Gemini) can operate the same canonical flow.

## Skill Invocation

The `sysdoc-analise` skill is triggered by:

```
sysdoc [pasta]
sysdoc [pasta] - [instrução extra]
sysdoc render [pasta]
```

When triggered, always read `skills/sysdoc/SKILL.md` first — it is the single source of truth for the operational flow.

## CLI Commands (Ferramentas Base)

A CLI do SysDoc é estritamente offline/determinística. Ela **não** faz chamadas de LLM. 

```bash
sysdoc status                           # list all projects and their state
sysdoc init [pasta]                     # create a new project folder structure
sysdoc prepare [pasta]                  # extract PDFs/DOCX → .sysdoc/cache/
sysdoc validate [pasta]                 # validate dados_consolidados.json
sysdoc render [pasta]                   # render HTML from dados_consolidados.json
sysdoc publish [pasta]                  # validate + version JSON + render HTML
sysdoc deploy [pasta]                   # find next index via SSH and SCP html to VPS
sysdoc compare [pasta]                  # compara versoes json geradas
python sysdoc_gui.py                    # launch tkinter GUI (offline features only)
```

## Macros de Prompt (Orquestrados pelo Agente)

O SysDoc delegou a orquestração de LLMs e chamadas de rede para o Agente de IA externo (você). Portanto, os seguintes comandos devem ser interpretados como "Macros" solicitados pelo usuário no chat:

1. **`sysdoc init [pasta]`**: O Agente cria a pasta base, orienta o usuário e aciona `sysdoc prepare`.
2. **`sysdoc all [pasta]`**: O Agente orquestra as fases:
   - **Fase 1 (Preparação)**: Roda `sysdoc prepare [projeto]` no bash.
   - **Fase 2 (Análise)**: Lê os arquivos de cache/textos, gera com a SUA própria LLM o arquivo `dados_consolidados.json` no disco.
   - **Fase 3 (Publicação)**: Roda `sysdoc publish [projeto]`. (Valida e gera HTML)
   - **Fase 4 (Deploy)**: Roda `sysdoc deploy [projeto]`. (Envia VPS)
3. **`sysdoc create TR [pasta]`**: O Agente cria o TR baseado nos dados.

Validate and render directly:
```bash
python templates/validate_sysdoc.py [pasta]/dados_consolidados.json [pasta]
python templates/render_analise.py [pasta]/dados_consolidados.json [pasta]
```

## Architecture

### Core files

| File | Role |
|---------|------|
| `skills/sysdoc/SKILL.md` | **Canonical operational flow** — IA-agnostic, single source of truth |
| `.claude/skills/sysdoc-analise/SKILL.md` | Thin Claude Code wrapper; delegates to canonical |
| `sysdoc.py` | CLI Entrypoint: `status`, `connect`, `models`, `init`, `prepare`, `analyze`, `validate`, `render`, `publish`, `deploy` |
| `run_sysdoc.sh` | Script wrapper que roda Fases 1, 2 e 4 (Prepare, Analyze, Deploy) sequencialmente |
| `sysdoc_gui.py` | tkinter GUI wrapping the same CLI subprocesses |
| `templates/validate_sysdoc.py` | Deterministic JSON validator (schema + coherence + PT-BR accents + traceability) |
| `templates/render_analise.py` | Deterministic HTML renderer — **immutable** |
| `templates/analise_template.html` | HTML template — **immutable** |
| `templates/schema_sysdoc.json` | Canonical JSON schema |
| `tests/test_validate.py` | Automated tests (pytest) — run before and after any change |
| `ROADMAP.md` | **Planned improvements** — read this before adding new features |
| `CHANGELOG.md` | Version history — update this after any change |

### Project folder structure

Each analysis project is a subdirectory containing:

```
[pasta]/
├── ETP.pdf                                    # required
├── TR.pdf                                     # required
├── modelos/ or Modelos/                       # required reference files (PDF, DOCX, TXT, MD)
├── .sysdoc/cache/
│   ├── contexto_sysdoc.md                     # deterministic token-saving map (generated by prepare)
│   ├── manifest.json
│   └── textos/ETP.txt, TR.txt, REF-*.txt
├── dados_consolidados.json                    # active analysis output
├── dados_consolidados_[model]_[date].json     # versioned on publish
└── analise_[model]_[date].html               # rendered report
```

`IGNORED_DIRS` in `sysdoc.py` (never treated as projects): `.git`, `.claude`, `backup`, `skills`, `templates`.

### Data flow

```
ETP.pdf + TR.pdf + modelos/
    → prepare → .sysdoc/cache/ (extracted text + contexto_sysdoc.md)
    → LLM analysis (reads SKILL.md + schema + contexto + extracted texts)
    → dados_consolidados.json
    → validate → render → analise_[model]_[date].html
```

The `analyze` command dynamically routes the prompt to the configured provider (OpenRouter, OpenAI, Gemini, or Anthropic) utilizing strict JSON outputs (`json_object` or `json_schema`) enforced via `schema_sysdoc.json`.

### Environment & Configuration

API Keys and provider preferences are centrally managed in `~/.sysdoc/config.json`, populated via the `sysdoc connect` and `sysdoc models` commands. Alternatively, standard environment variables are supported:

| Variable / Config | Purpose | Default |
|----------|---------|---------|
| `OPENROUTER_API_KEY` | Key for OpenRouter provider | — |
| `OPENAI_API_KEY` | Key for OpenAI provider | — |
| `GEMINI_API_KEY` | Key for Google Gemini provider | — |
| `ANTHROPIC_API_KEY` | Key for Anthropic provider | — |
| `SYSDOC_MAX_LLM_CONTEXT_CHARS` | Max chars fed to LLM | `700000` |

## Key Rules

- `templates/analise_template.html` and `templates/render_analise.py` are **immutable** — never edit them during analysis.
- Never write HTML manually — always use the renderer.
- `modelo_ia` in the JSON must be the real model slug (e.g., `claude-sonnet-4-6`, `gpt-5`, `gemini-2-5-pro`). Generic values like `ia`, `modelo`, `default` will fail validation.
- All generated fields must use correct Brazilian Portuguese (acentuação em norma culta). The `de` field is the only exception — it preserves the literal original text.
- Do not invent article numbers, SEI numbers, acórdãos, values, dates, or clauses. Use `pendente` when the basis is insufficient.
- Coherence rules enforced by the validator: `classificação=risco` → `risco_jurídico` ∈ {`relevante`, `bloqueante`}; `classificação=conforme` → `risco_jurídico=informativo`; `risco_jurídico=bloqueante` → `severidade` ∈ {`crítica`, `alta`}.

## PDF Extraction Priority (Claude Code)

1. `sysdoc prepare [pasta]` — generates deterministic cache and `contexto_sysdoc.md`
2. MCP `PDF_Tools` (`read_pdf_content`) — pre-authorized in `.claude/settings.local.json`
3. `pdftotext -layout` via Bash
4. `pandoc` for `.docx` references

Do not use image-based PDF tools on PDFs that have a text layer.
